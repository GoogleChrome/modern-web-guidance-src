- implement a passkey management interface inside `index.html` inside the container annotated with `data-testid="credential-list"`. Fetch the user's registered credentials from `/api/credentials` on `DOMContentLoaded` and render them, gracefully handling the zeroed AAGUID `00000000-0000-0000-0000-000000000000` by skipping the provider lookup and falling back safely. Each rendered item should display the credential name, a Rename button annotated with `data-testid="rename-button-${id}"` (which PUTs to `/api/credential/${id}`), and a Delete button annotated with `data-testid="delete-button-${id}"` (which DELETEs `/api/credential/${id}`). Display status in the element annotated with `data-testid="management-status"`. Keep the user's password manager in lockstep with the server: invoke the native `PublicKeyCredential.signalAllAcceptedCredentials()` Signal API automatically on page load with the full Base64URL-encoded credential ID list, and again immediately after every successful deletion with the updated list. Invoke `PublicKeyCredential.signalCurrentUserDetails()` immediately after every successful rename. Do NOT wrap the Signal API with third-party libraries — call the native browser APIs directly.
