Overflow Clipping Control

This demo showcases controlled clipping directly on replaced elements (images) using overflow: clip and overflow-clip-margin. Modern browsers natively default replaced elements to clip as of Chrome 108. Setting extended margins enables protecting inner padding frames from sub-pixel rendering leakage or allowing decorative ink overflow (drop-shadows) to remain visible without layout expansion.

Replaced Element (content-box)

Applies directly to an <img> with padding to create a clean gutter frame, preventing sub-pixel image leakage over the inner padding edge.

Replaced Element (Bleed Offset)

Applies overflow-clip-margin: 15px directly to an image to safely contain the element while allowing ink overflow (drop shadow) to remain visible.
