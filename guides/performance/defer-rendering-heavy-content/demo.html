Deferring Rendering Techniques

This demo compares different methods for deferring the rendering of complex UI components (like heavy dashboards) to improve initial page load performance and main-thread responsiveness.

Example 1: Toggle State (content-visibility vs display:none)

This dashboard is hidden by default using content-visibility: hidden. Unlike display: none, the browser parses the DOM but skips rendering. When toggled to visible, it renders instantly because the layout tree is preserved.

Example 2: Below the Fold (content-visibility: auto)

This dashboard is rendered below the fold. It uses content-visibility: auto. The browser completely skips layout and painting for this section until you scroll close to it, drastically improving the initial load time of the page!

Example 3: Searchable & Deferred (hidden="until-found")

This section uses hidden="until-found". Like Example 2, rendering is skipped, saving main-thread time on load. However, unlike Example 2, the text inside it is searchable via the browser's find-in-page (Cmd/Ctrl+F) mechanism.
