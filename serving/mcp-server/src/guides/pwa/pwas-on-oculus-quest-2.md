# PWAs on Oculus Quest&nbsp;2

The Oculus Quest&nbsp;2 is a virtual reality (VR) headset created by Oculus, a division of Meta. Developers can now build and distribute 2D and 3D Progressive Web Apps (PWA) that take advantage of Oculus Quest&nbsp;2's multitasking feature.

## Best Practices

### Web App Manifest Requirements

*   **DO** include `name` and `display` (either `"standalone"` or `"minimal-ui"`) as required fields in your Web App Manifest.
*   **DO** consider `short_name` for a concise app name.
*   **DO** specify `scope` and `start_url` for better control over the PWA's behavior, although they are optional on Oculus.
*   **DO** utilize proprietary fields like `ovr_package_name` for custom APK package naming and `ovr_multi_tab_enabled` to enable a tab bar within your PWA.
*   **DO** leverage `ovr_scope_extensions` to broaden the set of web pages considered part of the app's scope.

### Packaging PWAs

*   **DO** use tools like Bubblewrap CLI or PWABuilder for packaging PWAs for Meta Quest devices, as they simplify the process of generating APKs.
*   **DO** consider PWABuilder's option to use a modified Oculus Platform Utility for smaller APK sizes.
*   **DO** use the `--metaquest` flag with Bubblewrap CLI for Meta Quest compatibility.
*   **DO** use the Oculus Platform Utility for packaging if you specifically need an APK only compatible with Meta Quest devices and distributable through the Meta Quest Store.
*   **DO NOT** forget that APKs generated by Oculus Platform Utility are not compatible with regular Android devices.

### Installation and Debugging

*   **DO** enable Developer Mode on your Oculus device to allow sideloading via ADB.
*   **DO** use `adb install <your-app>.apk` or `bubblewrap install` to sideload your PWA.
*   **DO** leverage Chrome DevTools for debugging by connecting your Oculus device and inspecting the running PWA in Oculus Browser via `chrome://inspect/#devices`.
*   **DO** use tools like Android Web Toolbox for sideloading APKs directly from the browser using the Web USB API.

### App Discovery and Submission

*   **DO** utilize the Oculus Store and the browser itself for PWA discovery.
*   **DO** ensure your PWA is available in the Oculus Store for users to find and install.
*   **DO** note that `BeforeInstallPromptEvent` is not currently fired in Oculus Browser, despite feature detection suggesting support.
*   **DO** be aware that App Lab distribution for PWAs is not yet available, and submission to the Oculus Store requires approval as a Meta Quest Store developer.

### Immersive 3D WebXR PWAs

*   **DO** explore the WebXR API for building immersive 3D experiences.
*   **DO** use the WebXR Hand Input API and Meta's hand-tracking system for natural interactions in VR.
*   **DO** utilize WebXR Augmented Reality (AR) / Mixed Reality (MR) capabilities for passthrough experiences on compatible devices.
*   **DO** expect that entering VR mode requires user permission, and notifications are not currently supported within immersive mode.

## Fallback Strategies

*   **DO** implement feature detection for PWA-related APIs and polyfills where necessary to ensure compatibility across different environments.
*   **DO** refer to browser-specific documentation for Oculus Browser and Chrome for detailed feature support and potential limitations.
